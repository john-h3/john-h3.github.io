Java 中的对象

Java

对象从何而来

Object obj = new Object();
Object obj = Object.class.getDeclaredConstructor().newInstance();

上述的两个方法是比较常见的新建对象的方式，大部分书上会告诉我们第一种方式可以创建对象，但却不会告诉我们创建是怎样发生的。因此，对象从何而来我们不会知道，所以在遇到对象的初始化时，总会遇到各种各样的奇怪现象。

在不考虑类的加载情况下，两种构建对象的方式是等价的。由于第一种方式采取关键字，所以我们从第二种方式的代码来分析。首先看一下 newInstance() 。

public T newInstance(Object ... initargs) {
    ...
    ConstructorAccessor ca = constructorAccessor;   // read volatile
    if (ca == null) {
        ca = acquireConstructorAccessor();
    }
    T inst = (T) ca.newInstance(initargs);
    return inst;
}

关键的方法是 ca.newInstance() ，ca 是一个接口的具体实现，现在分析 ca 的创建，从 acquireConstructorAccessor() 中可以得到下面代码是创建方法。

public ConstructorAccessor newConstructorAccessor(Constructor<?> c) {
    ...
    NativeConstructorAccessorImpl acc = new NativeConstructorAccessorImpl(c);
    DelegatingConstructorAccessorImpl res = new DelegatingConstructorAccessorImpl(acc);
    acc.setParent(res);
    return res;
}

主要关注 NativeConstructorAccessorImpl ，尽管返回的是委托对象，但实际调用仍是 NativeConstructorAccessorImpl 。从 NativeConstructorAccessorImpl 中能发现最终调用的是一个本地方法。

对于对象在虚拟机中的具体创建可以参考这篇文章：对象的创建与内存分配 。

这里简单说下，本地方法会先为对象分配空间，然后再从上到下执行成员的初始化。

public class SomeObj {
    {
        name = "";
    }
    private final String name;
}

继承如何影响对象的构建

上面一节只是从对象自身出发，但是 Java 中有着继承，如果对象涉及到继承那么对象的创建就会复杂的多。大多数的书上会说明调用自身构造器之前，会递归调用基类的构造器。我们都知道 Java 中的所有类都继承自 Object ，那么构造一个对象时会递归创建它的基类对象吗？

我们可以为 newInstance 方法设置一个断点，然后查看构造一个对象的时候是否会发生调用两次 newInstance 方法。事实上并没有，这也就说明并不会递归地去创建基类对象。通过断点后的栈帧看来，本地方法会递归的将基类的成员都放入同一个对象之中。

先来看一段不会引发问题的代码：

public class Parent {
    String name = "parent";

    Parent() {
        System.out.println(name);
    }
}

public class Child extends Parent {
    String name = "child";

    Child() {
        System.out.println(name);
    }

    public static void main(String[] args) {
        new Child();
    }
}

输出结果：

parent
child

但是如果做一点小的修改：

public class Parent {
    String name = "parent";

    Parent() {
        printName();
    }

    void printName() {
        System.out.println(name);
    }
}

public class Child extends Parent {
    String name = "child";

    Child() {
        printName();
    }

    @Override
    void printName() {
        System.out.println(name);
    }

    public static void main(String[] args) {
        new Child();
    }
}

输出结果：

null
child

在第一次遇见这种情况时，我也是疑惑的，因为以我的理解第一个输出应该是 parent ，其实错了。因为曾经我也认为对子类对象的创建会同时创建父类对象，但现在我能确定只有一个对象被创建，那么一切也就能说通了。如果说重复的方法和成员都在一个类中，那是不可能实现的，那么编译器一定对这些同名加了修饰。逻辑上来说子类比父类更加具体，所以子类会覆盖重复的方法和变量，我们可以通过这么一段伪代码来理解这个复合类：

public class Child extends Parent {
    String super.name = "parent";
    void super.printName() {
        System.out.println(super.name);
    }
    String this.name = "child";
    void this.printName() {
        System.out.println(this.name);
    }
}

思考上面的伪代码，就应该不会再有错误的理解。
事实上，如果仔细思考过 this 与 super 关键字，那么就能直接得出伪代码中的结论。
你认为 this 是什么？ this 其实就是当前的对象的引用。
因为 this 和 super 太相似了，所以我曾经也直接认为 super 是父对象的引用，其实从这里就埋下了祸根。
下面是个验证：

Object child = this;
Object parent = super;

这段代码会在第二行报错，显然 super 不是对象，super 只是一个用来关联父类中的成员在子类对象中具体引用的特殊引用。
